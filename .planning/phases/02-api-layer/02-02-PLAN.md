**Part B: Create src/lib/api/treasury.ts**

Create the composite treasury module with these exports:

1. **`HoldingValuation` interface:**
```
{
  mint: string
  symbol: string
  name: string
  description?: string
  xAccount?: string
  bagsLink?: string
  uiAmount: number
  currentPriceUsd: number
  currentPriceSol: number
  currentValueUsd: number
  currentValueSol: number
  purchaseDate?: number
  purchasePriceUsd?: number
  costBasisUsd?: number
  gainLossUsd?: number
  gainLossPct?: number
  category: string
}
```

2. **`getTreasuryValuations(): Promise<TreasuryResponse>`** where TreasuryResponse is:
```
{
  solBalance: number
  solPriceUsd: number
  holdings: HoldingValuation[]
  totalValueUsd: number
  totalValueSol: number
}
```

Implementation steps:
a. Import `getSolBalance`, `getTokenAccountsByOwner` from `./helius`
b. Import `getMultiTokenPrices`, `getPriceAtTimestamp` from `./birdeye`
c. Import `getAccountTransfers` from `./solscan`
d. Import `TREASURY_WALLET`, `SOL_MINT` from `@/lib/constants`
e. Import `HOLDINGS_BY_MINT` from `@/lib/investments.config`

f. Fetch SOL balance + token accounts + SOL price in parallel using `Promise.all`
g. Filter to non-zero token accounts
h. Call `getMultiTokenPrices()` with all non-zero mints in ONE batch call
i. For each holding, in parallel (Promise.all):
   - Look up metadata from HOLDINGS_BY_MINT (symbol, name, description, xAccount, bagsLink)
   - For holdings without metadata: use mint as name, category as 'unknown'
   - Fetch first inbound transfer via `getAccountTransfers(TREASURY_WALLET, { token: mint, flow: 'in', page_size: 50 })` — last item in array is oldest = purchase date
   - If purchase date found, call `getPriceAtTimestamp(mint, purchaseDate)` for historical price
   - Compute gainLossUsd and gainLossPct from current vs purchase price

j. Compute totals (totalValueUsd, totalValueSol)
k. Wrap all per-holding fetches in try/catch — if historical data fails, still return current values with undefined gain/loss fields

CRITICAL: Use `Promise.all` for the per-holding enrichment loop (research pitfall #7 — sequential fetching causes 15+ second load times). The multi_price call already batches current prices. Historical lookups run in parallel per holding.

CRITICAL: Handle missing prices gracefully (research pitfall — Birdeye may not have data for illiquid tokens). Default to 0 for missing prices.

**Part A: Create src/lib/api/burns.ts**

Exports:

1. **`BurnTransaction` interface:** `{ txHash: string, timestamp: number, amount: number }` — amount is human-readable (divided by token decimals).

2. **`BurnSummary` interface:**
```
{
  totalBurned: number        // total $BRAIN burned, human-readable
  totalSupply: number        // current total supply, human-readable
  burnedPct: number          // % of original supply burned
  transactions: BurnTransaction[]
}
```

3. **`getBurnSummary(): Promise<BurnSummary>`**

Implementation:
a. Import `getAllTokenTransfers` from `./solscan`
b. Import `getTokenSupply` from `./helius`
c. Import `BRAIN_TOKEN_MINT`, `BURN_SOURCE`, `BURN_DESTINATION` from `@/lib/constants`

d. Fetch in parallel: `getAllTokenTransfers(BRAIN_TOKEN_MINT, { from: BURN_SOURCE, to: BURN_DESTINATION })` AND `getTokenSupply(BRAIN_TOKEN_MINT)`

e. Use `getAllTokenTransfers` (the paginated version) to get ALL burn txs, not just first page. Research open question #2 warns burns may span multiple pages.

f. Map transfers to BurnTransaction[]: divide `amount` by `10^token_decimals` (read decimals from each transfer's `token_decimals` field — do NOT hardcode, per research pitfall #6).

g. Compute totalBurned from sum of all transaction amounts.

h. Compute burnedPct: `totalBurned / (totalBurned + currentSupply) * 100` — burned tokens reduce circulating supply but we want % of original.

i. Sort transactions by timestamp descending (most recent first).

**Part B: Create src/lib/api/reflections.ts**

Exports:

1. **`ReflectionDistribution` interface:** `{ txHash: string, timestamp: number, amountSol: number, toAddress: string }`.

2. **`ReflectionSummary` interface:**
```
{
  totalDistributedSol: number
  distributions: ReflectionDistribution[]
}
```

3. **`getReflectionDistributions(): Promise<ReflectionSummary>`**

Implementation:
a. Import `getAllAccountTransfers` from `./solscan`
b. Import `LP_WALLET` from `@/lib/constants`

c. Fetch `getAllAccountTransfers(LP_WALLET, { flow: 'out' })` — all outbound SOL from LP wallet

d. Map transfers: divide amount by 1_000_000_000 (lamports to SOL). Include toAddress for tracking where distributions went.

e. Compute totalDistributedSol from sum.

f. Sort by timestamp descending.

4. **`getLpFeeInflows(): Promise<{ totalFeeSol: number; inflows: Array<{ txHash: string; timestamp: number; amountSol: number; fromAddress: string }> }>`**

Implementation:
a. Fetch `getAllAccountTransfers(LP_WALLET, { flow: 'in' })` — all inbound SOL to LP wallet (fee income)
b. Map, sum, sort same pattern as distributions
c. This function supports R8 (LP Fees) — total fees earned and fee inflows over time chart data

5. **`getFeeDistribution(totalFeeSol: number): Record<string, { pct: number; sol: number }>`**

Pure computation function — no API calls. Takes total fee SOL and returns breakdown per the 10/20/30/10/5/5/20 split:
- burned: 10%
- holders: 20%
- investments: 30%
- liquidity: 10%
- marketing: 5%
- dexBoosts: 5%
- dev: 20%

This supports R4 (Fee Distribution Ledger).
